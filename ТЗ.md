ТЕХНИЧЕСКОЕ ЗАДАНИЕ (АКТУАЛИЗАЦИЯ: ИЮНЬ 2026)
Модуль: Интерактивная карта-планировка коммерческих площадей (PropTech SaaS).

1. Актуальный стек технологий (Production-ready 2026)
Для реализации выбрана модульная экосистема, которая гарантирует моментальный рендеринг SVG на клиенте и отсутствие проблем с холодным стартом на бэкенде.

Фронтенд-стек (на выбор под архитектуру)
Фреймворк: Nuxt v4 (4.4+) (полная нативная поддержка TypeScript, оптимизированные Nitro-таски) или Next.js v16.2+ (на базе React 19).

Стилизация: Tailwind CSS v4.3+ (использует новый CSS-first конфиг @theme, встроенную стилизацию скроллбаров, логические свойства и компилируется в 4 раза быстрее старой v3).

Компоненты & Интерактив: Shadcn/ui (на базе Radix UI) + библиотека плавного зума @panzoom/panzoom или react-zoom-pan-pinch (v3.x) / vue-zoom-pan-pinch для бесшовного управления холстом без лагов производительности.

Бэкенд & Инфраструктура
Среда: Node.js v22+ (LTS) / v24 (нативная поддержка ESM модулей).

Фреймворк: NestJS v11.1+ (архитектура на ESM, поддержка валидации через Standard Schema и пайпы Zod/Valibot вместо устаревшего class-validator).

БД & ORM: PostgreSQL 16+ + Prisma ORM v7.7+.

Критически важно в 2026: Из Prisma 7 полностью удален тяжелый Rust-движок (Query Engine). Теперь компилятор работает как легковесный WebAssembly-модуль прямо в основном JS-потоке, что убирает проблемы с задержками и совместимостью платформ. Появилась нативная поддержка мультифайловых схем (prisma bootstrap).

2. Архитектурные требования: Чтобы не было «проёбов»
Главный проёб №1: Съезжание координат при ресайзе экрана
Решение: Запрещено хранить координаты полигонов в пикселях (px). Все точки кликов при разметке должны на лету конвертироваться во фракции (относительные величины от 0.0000 до 1.0000) относительно исходного разрешения загруженного изображения.

Формула записи: x_rel = event.offsetX / image_natural_width

Отрисовка на фронтенде: SVG рендерится с viewBox="0 0 1000 1000", а относительные координаты бэкенда масштабируются под эту сетку, обеспечивая идеальную адаптивность на смартфонах и 4K-мониторах.

Главный проёб №2: Падение производительности при 500+ офисах на этаже
Решение: Inline-рендеринг сырого SVG в DOM может перегрузить страницу. Для сложных планировок (например, больших ТРЦ) использовать Leaflet.js через L.imageOverlay. Но для стандартных БЦ (до 150 помещений на этаж) используем чистый SVG, оптимизированный через CSS-свойство content-visibility: auto и will-change: transform.

3. Спецификация модулей системы
Модуль А: Конструктор разметки (Панель Администратора)
Интерфейс: Холст с загруженным планом (JPEG/PNG) и верхний тулбар управления: [Инструмент "Полигон"], [Отмена точки], [Завершить обводку].

Логика отрисовки: * Клик левой кнопкой мыши вешает точку и рисует динамическую линию за курсором до следующего клика.

Клик на самую первую точку (или нажатие Enter) замыкает контур.

Интеграция с БД: При замыкании открывается модалка Shadcn. Менеджер вводит ID/номер помещения. На бэкенд летит Payload:

Рендеринг: Фронтенд запрашивает схему этажа. Компонент карты отрисовывает подложку и циклом выводит SVG <polygon> элементы.

4. Схема базы данных (Prisma v7.7 Спецификация)
Используем явное разделение на сущности. Хранение координат заворачиваем в тип данных Json внутри PostgreSQL.

Фрагмент кода
// file: schema.prisma

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
  // В Prisma 7 новые оптимизации компиляции включаются по умолчанию
}

enum RoomStatus {
  FREE
  BOOKED
  OCCUPIED
  OVERDUE
  REPAIR
}

enum InvoiceStatus {
  UNPAID
  PAID
  CANCELLED
}

model Building {
  id        String   @id @default(uuid())
  name      String
  address   String
  floors    Floor[]
  createdAt DateTime @default(now())
}

model Floor {
  id            String   @id @default(uuid())
  buildingId    String
  building      Building @relation(fields: [buildingId], references: [id], onDelete: Cascade)
  floorNumber   Int
  planImageUrl  String   // Ссылка на S3 / CDN хранилище
  rooms         Room[]
}

model Room {
  id                 String     @id @default(uuid())
  floorId            String
  floor              Floor      @relation(fields: [floorId], references: [id], onDelete: Cascade)
  roomNumber         String
  area               Float      // Площадь в кв.м.
  basePrice          Decimal    // Базовая ставка аренды
  polygonCoordinates Json       // Массив [{x: Float, y: Float}]
  currentStatus      RoomStatus @default(FREE)
  contracts          Contract[]
}

model Tenant {
  id          String     @id @default(uuid())
  companyName String
  email       String     @unique
  phone       String
  contracts   Contract[]
}

model Contract {
  id         String    @id @default(uuid())
  roomId     String
  room       Room      @relation(fields: [roomId], references: [id])
  tenantId   String
  tenant     Tenant    @relation(fields: [tenantId], references: [id])
  startDate  DateTime
  endDate    DateTime
  isActive   Boolean   @default(true)
  invoices   Invoice[]
}

model Invoice {
  id         String        @id @default(uuid())
  contractId String
  contract   Contract      @relation(fields: [contractId], references: [id], onDelete: Cascade)
  amount     Decimal
  dueDate    DateTime      // Крайний срок оплаты
  status     InvoiceStatus @default(UNPAID)
  createdAt  DateTime      @default(now())
}
